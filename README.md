# Run and deploy TOKIN Smart Room

This contains everything you need to run your app locally.


## Run Locally

**Prerequisites:**  Node.js


1. Install dependencies:
   `npm install`
2. For local verification email testing, copy `functions/.env.example` to `functions/.env` and set `POWER_AUTOMATE_VERIFICATION_FLOW_URL` to the Power Automate HTTP trigger URL. Keep `APP_BASE_URL=https://tokinsmartroom-495306.web.app` so test emails use the production verification page.
3. Run the app:
   `npm run dev`

For LAN access, open the app from the host PC IP and port, for example `http://192.168.x.x:3000`. Booking verification emails still use `https://tokinsmartroom-495306.web.app`, not the LAN origin.

## Verification Email Configuration

The Cloud Functions backend sends booking verification emails through Power Automate. Production must store the flow URL as a Firebase Functions secret:

```bash
firebase functions:secrets:set POWER_AUTOMATE_VERIFICATION_FLOW_URL --project sutsmartbus-495306
firebase deploy --only functions --project sutsmartbus-495306
```

Before deploying, temporarily move or remove `functions/.env`. Firebase CLI reads that file during deploy, and keys that are also declared with `defineSecret(...)` will conflict with the Firebase Functions secret bindings.

Verification links generated by Cloud Functions use `APP_BASE_URL`, with `https://tokinsmartroom-495306.web.app` as the canonical TOKIN Smart Room fallback. Local, IP-based, non-HTTPS, and legacy app URLs are ignored for verification email links. Do not use legacy Hosting URLs such as `https://sutsmartbus-495306.web.app` or `https://tokinsmartroom.web.app` for verification emails.

Do not commit the real Power Automate URL. `functions/.env` is ignored by git and is only for local development.
