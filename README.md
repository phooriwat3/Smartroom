# Run and deploy TOKIN Smart Room

This contains everything you need to run your app locally.


## Run Locally

**Prerequisites:**  Node.js


1. Install dependencies:
   `npm install`
2. For local verification email testing, copy `functions/.env.example` to `functions/.env` and set `POWER_AUTOMATE_VERIFICATION_FLOW_URL` to the Power Automate HTTP trigger URL. Keep `APP_BASE_URL=https://tokinsmartroom-495306.web.app` unless you intentionally want local test emails to point at your local dev server.
3. Run the app:
   `npm run dev`

For LAN access, open the app from the host PC IP and port, for example `http://192.168.x.x:3000`. Booking verification emails use the same LAN origin that created the booking.

## Verification Email Configuration

The Cloud Functions backend sends booking verification emails through Power Automate. Production must store the flow URL as a Firebase Functions secret:

```bash
firebase functions:secrets:set POWER_AUTOMATE_VERIFICATION_FLOW_URL --project sutsmartbus-495306
firebase deploy --only functions --project sutsmartbus-495306
```

Verification links generated by Cloud Functions use `APP_BASE_URL`, with `https://tokinsmartroom-495306.web.app` as the canonical TOKIN Smart Room fallback. Do not use legacy Hosting URLs such as `https://sutsmartbus-495306.web.app` or `https://tokinsmartroom.web.app` for verification emails.

Do not commit the real Power Automate URL. `functions/.env` is ignored by git and is only for local development.
